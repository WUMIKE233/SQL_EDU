# 数据库基础与环境准备

## 1.1 什么是数据库

数据库是按照一定结构组织、存储和管理的数据集合。与普通文件相比，数据库更适合处理：

- 大量结构化数据
- 多人共享的数据
- 频繁查询与更新的数据
- 需要一致性与安全控制的数据

数据库管理系统（DBMS）是帮助我们管理数据库的软件，例如 PostgreSQL、MySQL、SQL Server 和 Oracle。

## 1.2 关系数据库的核心概念

关系数据库使用二维表组织数据。

- 表（Table）：同类数据的集合，如学生表
- 行（Row / Tuple）：一条记录，如某位学生的信息
- 列（Column / Attribute）：某个字段，如姓名、学号
- 主键（Primary Key）：唯一标识一行数据的列或列组合
- 外键（Foreign Key）：用于表示表与表之间的关联

例如，学生表可以表示为：

| student_id | student_name | major | enroll_year |
| --- | --- | --- | --- |
| 2023001 | 张三 | 计算机科学 | 2023 |
| 2023002 | 李四 | 信息管理 | 2023 |

## 1.3 SQL 的作用

SQL（Structured Query Language）是结构化查询语言，用于：

- 定义数据库对象，如库、表、视图
- 操作数据，如插入、删除、更新
- 查询数据，如筛选、排序、连接、统计
- 控制权限与事务

常见分类如下：

- DDL：数据定义语言，如 `CREATE`、`ALTER`、`DROP`
- DML：数据操作语言，如 `INSERT`、`UPDATE`、`DELETE`
- DQL：数据查询语言，通常指 `SELECT`
- DCL：数据控制语言，如 `GRANT`、`REVOKE`
- TCL：事务控制语言，如 `COMMIT`、`ROLLBACK`

## 1.4 安装与工具准备

推荐使用 PostgreSQL 作为教学环境，原因包括：

- 免费开源
- 对标准 SQL 支持较好
- 功能完整，适合从入门到进阶

建议准备以下工具之一：

- pgAdmin：适合图形化管理
- DBeaver：跨数据库，操作方便
- DataGrip：适合 IDE 型使用场景
- `psql`：适合命令行练习

## 1.5 第一条 SQL

```sql
SELECT 'Hello, SQL!' AS message;
```

这条语句返回一个只有一行一列的结果表。尽管很简单，但它说明了一件重要的事：SQL 的输出本质上也是表。

## 1.6 本章小结

- 数据库用于组织和管理结构化数据
- 关系数据库以表为核心组织方式
- SQL 不只是“查询语言”，还承担定义、更新、控制等任务
- 学 SQL 时要同时关注“数据结构”和“查询逻辑”
