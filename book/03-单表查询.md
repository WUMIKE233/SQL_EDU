# 单表查询

## 3.1 `SELECT` 的基本结构

最常见的查询语句是：

```sql
SELECT column_list
FROM table_name
WHERE condition
ORDER BY column_name;
```

并不是每次都必须写全，但理解这个结构很重要。

## 3.2 查询全部列与指定列

```sql
SELECT * FROM students;
```

```sql
SELECT student_id, student_name, major
FROM students;
```

教学中建议少用 `SELECT *`，原因包括：

- 可读性较差
- 列较多时影响性能
- 表结构变动后可能影响程序结果

## 3.3 条件筛选

```sql
SELECT student_name, major
FROM students
WHERE enroll_year = 2023;
```

常见条件运算：

- 比较：`=`, `<>`, `>`, `<`, `>=`, `<=`
- 逻辑：`AND`, `OR`, `NOT`
- 范围：`BETWEEN ... AND ...`
- 集合：`IN (...)`
- 模糊：`LIKE`
- 空值：`IS NULL`, `IS NOT NULL`

## 3.4 模糊查询

```sql
SELECT student_name
FROM students
WHERE student_name LIKE '张%';
```

其中：

- `%` 表示任意长度字符
- `_` 表示单个字符

## 3.5 排序与限制结果

```sql
SELECT student_id, student_name, enroll_year
FROM students
ORDER BY enroll_year DESC, student_id ASC;
```

```sql
SELECT student_name, gpa
FROM students
ORDER BY gpa DESC
LIMIT 5;
```

`ORDER BY` 决定输出顺序，`LIMIT` 用于截取部分结果，在做排行榜和分页时很常见。

## 3.6 计算列与别名

```sql
SELECT
    student_name,
    gpa,
    gpa * 25 AS score_percent
FROM students;
```

别名 `AS score_percent` 可以提升结果可读性。SQL 中很多查询并不是直接取值，而是“边算边查”。

## 3.7 去重

```sql
SELECT DISTINCT major
FROM students;
```

`DISTINCT` 用于去重，但如果经常依赖它“补救脏数据”，说明表设计或查询逻辑可能需要重新检查。

## 3.8 空值的理解

空值 `NULL` 不等于 0，也不等于空字符串。它表示“未知”或“不存在”。

错误写法：

```sql
SELECT * FROM students WHERE email = NULL;
```

正确写法：

```sql
SELECT * FROM students WHERE email IS NULL;
```

## 3.9 本章小结

- 单表查询是 SQL 学习的起点
- `WHERE` 用于过滤，`ORDER BY` 用于排序，`LIMIT` 用于截取结果
- 要区分“值为空”和“值未知”
- `SELECT` 的目标不只是拿数据，也包括组织和表达信息
