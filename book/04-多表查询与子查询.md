# 多表查询与子查询

## 4.1 为什么需要多表查询

真实系统中，数据通常分散在多张表里。比如：

- 学生信息在 `students`
- 课程信息在 `courses`
- 选课信息在 `enrollments`

如果想知道“某个学生选了哪些课程”，就必须把这些表关联起来。

## 4.2 内连接

```sql
SELECT
    s.student_name,
    c.course_name,
    e.score
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id;
```

这条语句表达的是：

- 从选课表出发
- 根据学号找到学生
- 根据课程号找到课程
- 最后输出学生名、课程名和成绩

## 4.3 左连接

```sql
SELECT
    s.student_name,
    e.course_id
FROM students s
LEFT JOIN enrollments e ON s.student_id = e.student_id;
```

`LEFT JOIN` 会保留左表的所有记录。即使某个学生暂时没有选课，也会被显示出来。

适用问题：

- 哪些学生尚未选课？
- 哪些教师当前没有授课？

## 4.4 多表连接时的注意点

- 总是写清连接条件，避免笛卡尔积
- 尽量给表起别名，如 `students s`
- 输出列名前加表别名，减少歧义

错误示例：

```sql
SELECT *
FROM students, courses;
```

这会得到每个学生和每门课程的所有组合，通常不是我们想要的结果。

## 4.5 子查询

子查询是嵌套在另一条 SQL 语句中的查询。

```sql
SELECT student_name
FROM students
WHERE department_id = (
    SELECT department_id
    FROM departments
    WHERE department_name = '计算机学院'
);
```

这条语句先查出“计算机学院”的院系编号，再查这个院系下的学生。

## 4.6 `IN` 与相关子查询

```sql
SELECT course_name
FROM courses
WHERE teacher_id IN (
    SELECT teacher_id
    FROM teachers
    WHERE title = '教授'
);
```

如果子查询会返回多行结果，通常可以配合 `IN` 使用。

## 4.7 `EXISTS`

```sql
SELECT s.student_name
FROM students s
WHERE EXISTS (
    SELECT 1
    FROM enrollments e
    WHERE e.student_id = s.student_id
);
```

`EXISTS` 常用于判断“是否存在相关记录”，在很多场景下可读性很好。

## 4.8 本章小结

- 多表查询是关系数据库的核心能力
- `JOIN` 解决“如何把分散的数据拼起来”
- 子查询解决“先算出一个条件，再进行筛选”
- 应根据问题选择连接、子查询或二者结合
