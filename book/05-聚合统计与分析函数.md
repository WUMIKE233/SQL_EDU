# 聚合统计与分析函数

## 5.1 为什么要做统计分析

SQL 不仅能“列出数据”，还能“总结数据”。在教学管理系统中，常见问题包括：

- 每个院系有多少学生？
- 每门课程的平均分是多少？
- 哪些课程选课人数超过 50？

这些都需要聚合统计。

## 5.2 常见聚合函数

| 函数 | 作用 |
| --- | --- |
| `COUNT()` | 计数 |
| `SUM()` | 求和 |
| `AVG()` | 平均值 |
| `MAX()` | 最大值 |
| `MIN()` | 最小值 |

示例：

```sql
SELECT COUNT(*) AS student_count
FROM students;
```

## 5.3 分组统计

```sql
SELECT
    department_id,
    COUNT(*) AS student_count
FROM students
GROUP BY department_id;
```

`GROUP BY` 的含义是：按某些列把数据分组，再对每组做统计。

注意：

- 出现在 `SELECT` 中但不是聚合函数的列，通常必须出现在 `GROUP BY` 中

## 5.4 分组后的筛选

```sql
SELECT
    course_id,
    AVG(score) AS avg_score
FROM enrollments
GROUP BY course_id
HAVING AVG(score) >= 85;
```

这里的 `HAVING` 作用于分组后的结果，而 `WHERE` 作用于分组前的原始行。

可以这样理解：

- `WHERE` 先筛行
- `GROUP BY` 再分组
- `HAVING` 最后筛组

## 5.5 综合示例

```sql
SELECT
    c.course_name,
    COUNT(e.student_id) AS selected_count,
    AVG(e.score) AS avg_score
FROM courses c
LEFT JOIN enrollments e ON c.course_id = e.course_id
GROUP BY c.course_id, c.course_name
ORDER BY selected_count DESC;
```

这条语句可以用于分析课程热度和课程成绩。

## 5.6 窗口函数入门

窗口函数不会把多行压缩成一行，而是在保留原始行的同时增加分析结果。

```sql
SELECT
    student_id,
    course_id,
    score,
    RANK() OVER (PARTITION BY course_id ORDER BY score DESC) AS rank_in_course
FROM enrollments;
```

这个查询能得到“每门课程内每个学生的成绩排名”。

常见窗口函数：

- `RANK()`
- `DENSE_RANK()`
- `ROW_NUMBER()`
- `SUM() OVER (...)`
- `AVG() OVER (...)`

## 5.7 本章小结

- 聚合函数用于从多条记录中提炼统计结果
- `GROUP BY` 是分组统计的核心
- `HAVING` 用于筛选分组后的结果
- 窗口函数适合做排名、累计和、分组比较等分析任务
