# 数据更新与事务

## 6.1 插入数据

```sql
INSERT INTO departments (department_id, department_name, office_phone)
VALUES (1, '计算机学院', '010-10000001');
```

一次插入多行：

```sql
INSERT INTO departments (department_id, department_name, office_phone)
VALUES
    (2, '数学学院', '010-10000002'),
    (3, '外国语学院', '010-10000003');
```

## 6.2 更新数据

```sql
UPDATE students
SET major = '数据科学'
WHERE student_id = 2023001;
```

若省略 `WHERE`，整张表都会被更新，因此在执行前必须反复确认条件。

## 6.3 删除数据

```sql
DELETE FROM enrollments
WHERE student_id = 2023001
  AND course_id = 101;
```

`DELETE` 删除的是行，不会删除表结构。

## 6.4 什么是事务

事务（Transaction）是一组要么全部成功、要么全部失败的操作。

典型场景：

- 学生退课时，需要同时删除选课记录并更新课程人数
- 转专业时，需要更新学生信息并记录日志

如果操作做到一半失败，就会产生不一致数据。事务就是为了解决这个问题。

## 6.5 事务控制

```sql
BEGIN;

UPDATE courses
SET selected_count = selected_count + 1
WHERE course_id = 101;

INSERT INTO enrollments (student_id, course_id, semester, score)
VALUES (2023001, 101, '2025-Fall', NULL);

COMMIT;
```

如果中途出错，可以：

```sql
ROLLBACK;
```

## 6.6 ACID 特性

事务通常具有四个经典性质：

- 原子性（Atomicity）：要么全做，要么全不做
- 一致性（Consistency）：事务前后数据规则不被破坏
- 隔离性（Isolation）：并发事务互不干扰
- 持久性（Durability）：提交后的结果不会丢失

## 6.7 并发问题直觉理解

多个用户同时访问数据库时，可能出现：

- 脏读
- 不可重复读
- 幻读
- 丢失更新

本科阶段不要求一开始就完全掌握细节，但要知道：事务并不只是“写个 `COMMIT`”，它与并发控制密切相关。

## 6.8 本章小结

- `INSERT`、`UPDATE`、`DELETE` 是最基本的数据更新操作
- 事务帮助我们维护复杂操作的一致性
- 任何会影响多张表或多条记录的业务，都应优先考虑事务
