# 索引、视图与安全

## 8.1 索引的作用

索引可以理解为数据库中的“目录”。如果没有索引，数据库可能需要逐行扫描整张表来寻找目标数据。

适合建索引的列通常包括：

- 经常出现在 `WHERE` 条件中的列
- 经常用于连接的外键列
- 经常用于排序和分组的列

示例：

```sql
CREATE INDEX idx_students_department_id
ON students(department_id);
```

## 8.2 索引不是越多越好

索引会带来额外成本：

- 占用存储空间
- 插入、更新、删除时需要维护索引

因此，索引应服务于核心查询，而不是盲目添加。

## 8.3 视图

视图是由查询结果定义出的“虚拟表”。

```sql
CREATE VIEW student_course_scores AS
SELECT
    s.student_id,
    s.student_name,
    c.course_name,
    e.score
FROM enrollments e
JOIN students s ON e.student_id = s.student_id
JOIN courses c ON e.course_id = c.course_id;
```

视图的常见作用：

- 简化复杂查询
- 对外屏蔽底层表结构细节
- 为不同角色提供不同的数据视角

## 8.4 权限控制

数据库通常允许不同用户拥有不同权限。

例如：

```sql
GRANT SELECT ON student_course_scores TO teaching_assistant;
```

```sql
REVOKE UPDATE ON students FROM teaching_assistant;
```

这样可以避免无关人员误改核心数据。

## 8.5 SQL 注入的基本认识

如果程序将用户输入直接拼接成 SQL，可能带来安全风险。

危险示例：

```text
SELECT * FROM users WHERE username = '" + input + "';
```

更安全的做法是：

- 使用参数化查询
- 最小化数据库权限
- 对输入做边界校验

## 8.6 本章小结

- 索引的目标是提升查询效率
- 视图可以简化查询并隔离底层结构
- 安全不是附加功能，而是数据库使用的基本要求
