# 综合实验与课程项目

## 9.1 实验目标

通过一个完整的小型项目，将前面章节的知识串联起来。学生完成本项目后，应能够：

- 根据需求设计关系表
- 编写建表脚本和初始化数据脚本
- 实现常见查询、统计与更新逻辑
- 使用事务保证关键业务一致性

## 9.2 项目题目

请设计并实现一个“小型校园教学管理系统”数据库。

### 基本要求

- 至少包含 5 张表
- 必须体现 1 对多关系和多对多关系
- 必须设置主键、外键和至少 2 类其他约束
- 至少编写 10 条有意义的查询语句
- 至少编写 2 条聚合统计语句
- 至少设计 1 个事务场景

## 9.3 建议任务拆分

1. 需求分析
2. 概念结构设计
3. 逻辑表结构设计
4. 编写 `CREATE TABLE`
5. 插入测试数据
6. 编写基础查询
7. 编写统计查询
8. 编写更新与事务脚本
9. 撰写实验报告

## 9.4 可选扩展方向

- 增加教室、排课、成绩复核等模块
- 引入奖学金、课程评价等实体
- 增加用户、角色和权限设计
- 尝试为高频查询设计索引并比较性能

## 9.5 实验报告建议结构

- 项目背景
- 需求分析
- E-R 模型说明
- 表结构设计
- 核心 SQL 展示
- 查询结果截图
- 遇到的问题与解决方法
- 课程总结

## 9.6 评分建议

| 维度 | 比例 |
| --- | --- |
| 结构设计合理性 | 25% |
| SQL 正确性 | 25% |
| 查询复杂度与完整性 | 20% |
| 事务与约束使用 | 15% |
| 报告表达与规范性 | 15% |

## 9.7 结语

SQL 的学习并不止于“会写语句”，更重要的是学会如何把现实世界的问题抽象成可管理、可验证、可分析的数据结构。只要持续练习，数据库会从“语法课”变成你理解信息系统的重要工具。
